C++智能指针

原理及实现
 2021.5.19  2021.5.27  C++  2918  6 分钟  

简介

C++中具有三个智能指针类，分别是shared_ptr，unique_ptr，weak_ptr，还有一个辅助类：enable_shared_from_this。

这三个类对于持有资源的方式如下表所示：

* shared_ptr unique_ptr weak_ptr
资源共享 Y N X
资源独占 N Y X
不占据资源 X X Y

智能指针是充分利用C++ RAII特性的一个良好示例，自行设计实现智能指针也是学习RAII机制的最佳方式。

独占资源

unique_ptr可以说是最容易实现的智能指针了，因为它不需要与其它指针共享资源，因此简单的禁用拷贝操作，就可以轻松实现。

template <typename T, typename Deleter = std::default_delete<T>>
class unique_ptr {
public:
    using element_type = T;
    using pointer = T*;

    unique_ptr() noexcept{}
    unique_ptr(std::nullptr_t) noexcept;
    explicit unique_ptr(pointer ) noexcept;
    unique_ptr(pointer , const Deleter& ) noexcept;
    unique_ptr(unique_ptr&& ) noexcept ;
    unique_ptr(const unique_ptr&) = delete;
    ~unique_ptr() noexcept ;

    unique_ptr& operator=(const unique_ptr) = delete;
    unique_ptr& operator=(unique_ptr&& ) ;
    unique_ptr& operator=(std::nullptr_t) noexcept ;

    element_type& operator*() const ;
    pointer operator->() const noexcept;
    explicit operator bool() const noexcept ;

    pointer release() noexcept ;
    pointer get() const noexcept;
    Deleter& get_deleter() noexcept ;
    const Deleter& get_deleter() const noexcept ;
};

template <typename T, typename Deleter>
class unique_ptr<T[], Deleter> {
public:
    using element_type = T;
    using pointer = T*;

    unique_ptr() noexcept;
    unique_ptr(std::nullptr_t) noexcept;
    explicit unique_ptr(pointer) noexcept ;
    unique_ptr(unique_ptr&&) noexcept ;
    unique_ptr(const unique_ptr&) = delete;
    ~unique_ptr() noexcept;

    unique_ptr& operator=(const unique_ptr&) = delete;
    unique_ptr& operator=(unique_ptr&&) ;
    unique_ptr& operator=(std::nullptr_t) noexcept ;

    explicit operator bool() const noexcept ;
    element_type& operator[](std::size_t) const ;

    pointer release() noexcept;
    pointer get() const noexcept;
    Deleter& get_deleter() noexcept;
    const Deleter& get_deleter() const noexcept;
};

这个类较容易实现，唯一需要关注的是删除器是否应该具有状态。也就是说：模板类是否应该持有删除器的实例。

C++规定如果不提供自定义的删除器，那么sizeof(unique_ptr) == sizeof(T*)，否则模板类的实例大小主要取决于删除器保存的状态。

为了搞成这个效果，需要使用Impl手段，还需要利用C++的一个规定： 允许作为基类子对象时为零大小

共享资源

为了共享资源，需要考虑如下的几个条件：

  • 如何判断资源已经无人持有了
  • 如何打破循环引用 (weak_ptr)
  • 如何从this指针共享资源 (enable_shared_from_this)
  • ......

下面来解决上面的几个问题。

判断资源已经无人持有了

使用一个变量来统计当前资源的引用数量，当变为0的时候，销毁资源。

变量是static吗？

不可以，考虑下面的情况：

1
2

shared_ptr<int> p(new int(5));
shared_ptr<int> q(new int(7));

对于上面的情况，变量的值为2，而实际上这是两类资源，不应该共享。

注：也有可能为1，取决于你如何实现，但无论如何实现，都将会使得该变> 量在p，q之间共享，有可能造成资源计数错误。

为了能够实现资源共享，并且不让引用计数在不同资源间共享，可以考虑实现一个控制块：当资源共享时，在控制块上记录信息，并且共享相同资源的对象也共享相同的控制块！

实现如下：

template <typename T>
struct control_block {
    T* data_;                          // 数据
    long ref_;                         // 引用计数
    std::function<void(T*)> deleter_;  // 删除器

    control_block(T* data, long ref);
    control_block(T* data, long ref, std::function<void(T*)> d);
};

什么时候创建控制块？

  • std::make_shared总会创建一个控制块。
  • 通过一个独享所有权的指针（如std::unique_ptr或std::auto_ptr）创建出的std::shared_ptr总会创建一个控制块。
  • 通过裸指针创建的std::shared_ptr会创建控制块。

打破循环引用 (weak_ptr)

考虑如下代码：

// 示例代码，不一定符合语法规则
struct Node {
    shared_ptr<Node> next;
    shared_ptr<Node> parent;
};

shared_ptr<Node> p1(new Node);
shared_ptr<Node> p2(new Node);
p1->next = p2;  // p2 引用计数为2
p2->parent = p1;  // p1 引用计数为2
// Now: p1->p2->p1

当p1析构的时候，p1的引用计数减1（2 -> 1）。 当p2析构的时候，p2的引用计数减1（2 -> 1）。

现在，p1和p2的引用计数都不为0，无法回收他们持有的内存，造成内存泄漏。

这里的问题在于，引用计数出现了循环：为了释放A，必须先释放B，而为了释放B，又必须先释放A。

为了解决这个问题，可以引入一个新的变量，只用来计数，不持有资源。

修改后的控制块如下：

template <typename T>
struct control_block {
    T* data_;                          // 数据
    long ref_;                         // 引用计数
    long weak_;                        // weak 计数
    std::function<void(T*)> deleter_;  // 删除器

    control_block(T* data, long ref);
    control_block(T* data, long ref, std::function<void(T*)> d);
};

weak_变量用来标记那些不持有资源的指针。

shared_ptr<Node> p1(new Node);

p2->parent = p1;  // p1 引用计数为1
// Now: p1 -> p2 ~> p1

当p1析构的时候，p1的引用计数减1（1 -> 0），p1释放资源，触发p1->next析构，p2的引用计数减1（2 -> 1）。

当p2析构的时候，p2的引用计数减1（1 -> 0）。p2释放资源

从this指针共享资源 (enable_shared_from_this)

考虑下面的情况：

class QP {
public:
	std::shared_ptr<QP> getptr() {
		return std::shared_ptr<QP>(this);
	}
};

std::shared_ptr<QP> p(new QP); // 引用计数为1
std::shared_ptr<QP> r = p->getptr();  // 引用计数为1
// delete twice.

注意：p和r指向的并不是同一个资源，因为他们从不同的裸指针构造，会各自创建一个控制块。

以上的代码出现的弊端就是会删除同一个指针两次，导致进程异常。

这里的问题在于，从对象内部的this指针构造的shared_ptr和该对象本身是脱离的，两者无法感知到对方的状态。

C++为了解决这个问题，引入了enable_shared_from_this。下面将讲述它的原理。

首先，为了使用enable_shared_from_this，需要将上面的代码更改如下：

class QP: public enable_shared_from_this<QP> {
public:
	std::shared_ptr<QP> getptr() {
		return shared_from_this();
	}
};

std::shared_ptr<QP> p(new QP); 
std::shared_ptr<QP> r = p->getptr();
// no error.

enable_shared_from_this的使用方法在C++中称为奇异递归模板模式 CRTP，因为父类的模板参数是子类。

在enable_shared_from_this内部保存了一个weak_ptr，这样enable_shared_from_this可以知道他在追踪哪个资源，同时还不会出现循环引用。

重要的是，由于继承关系的存在，这导致在执行std::shared_ptr<QP> p(new QP); 代码的时候，shared_ptr的构造函数可以将QP*向上转型为enable_shared_from_this*，进而给enable_shared_from_this内部的weak_ptr变量赋值。这时，就可以通过shared_from_this函数直接返回weak_ptr了。

template <typename T>
class weak_ptr {
public:
    ~weak_ptr() noexcept {
        if (cb_) {
            cb_->weak_--;
            if (cb_->weak_ <= 0 && cb_->ref_ <= 0) delete cb_;
        }
    }

    shared_ptr<T> lock() const noexcept {
        if (sptr_)
            return *sptr_;
        else
            return nullptr;
    }

private:
    const shared_ptr<T>* sptr_; // 也可以不保存，直接使用cb_构造一个新的shared_ptr
    control_block<T>* cb_;
};

template <typename T>
class enable_shared_from_this {
public:
    constexpr enable_shared_from_this() noexcept {}
    enable_shared_from_this(const enable_shared_from_this<T>&) noexcept {}

    enable_shared_from_this<T>& operator=(const enable_shared_from_this<T>& other) noexcept { return *this; }

    ~enable_shared_from_this() {}
    shared_ptr<T> shared_from_this() { return ptr_.lock(); }
    shared_ptr<const T> shared_from_this() const { return ptr_.lock(); }

private:
    weak_ptr<T> ptr_;

    friend shared_ptr<T>;
};

template <typename T>
class shared_ptr {
public:
    using element_type = T;
    // Example
    explicit shared_ptr(element_type* ptr) : data_(ptr), cb_(new control_block<element_type>(data_, 1, 0)) {
        if constexpr (std::is_base_of<enable_shared_from_this<T>, T>::value) {
            if (ptr) {
                enable_shared_from_this<T>& base = *ptr;
                base.ptr_ = *this;
            }
        }
    }
    // ...more
    ~shared_ptr() noexcept {
        cb_->ref_--;
        if (cb_->ref_ <= 0) {
            if (cb_->weak_ <= 0) {
                delete cb_;
                cb_ = nullptr;
            }

            if (cb_ && cb_->deleter_)
                cb_->deleter_(data_);
            else
                delete data_;
        }
    }

private:
    element_type* data_;  // 数据
    control_block<element_type>* cb_;

    friend weak_ptr<T>;
};

这里判断参数类型T是否继承了enable_shared_from_this<T>，如果继承了，那么向上转型，之后在父类里设置weak_ptr。

总结

智能指针很好用，但是必须时刻记住不要构成循环引用。因此，类与类的关系不要嵌套太深。另外，尽可能不要混用裸指针和智能指针，否则下面的代码迟早会碰见：

1
2
3

int *p = new int(7);
shared_ptr<int> p1(p);
delete p;
