SAT可以用来回答命题逻辑中的一些可满足性问题，例如：$P \vee Q \wedge R$

但是实际现实中的问题却是这样的：$ (a + b < c) \wedge (c + k > d) \vee (c = f(a) + b)$，这样的问题的可满足性该如何回答？

从逻辑上来看，这里出现了很多逻辑系统中不包含的符号，为了解决这类问题，必须首先知道他们的意思。

因此，引入术语理论（Theory）：

  • 理论用于对符号赋予含义。
  • 理论定义一系列的公理，以及使用公理可以推导出的结论。

粗略的看，一个理论就是构建了一个完整的系统，该系统是特化的，只为用来解决某些问题。从这个角度来看，逻辑更像是一种模板，它抽象出了一套核心，只要有需要，就可以重新解释逻辑，形成一个新的理论。

SMT

可满足性模理论（Satisfiability Modulo Theories，SMT），是指给定一组理论，根据给定背景逻辑，求在该组理论解释下公式的可满足性。

SMT = SAT + Theory

比如说：EUF（Equality with Uninterpreted Functions）

下面的命题可以满足吗？

$$ a = b \wedge b = (c - 2) \wedge A[f(a)] = A[f(c-2)] $$

首先应用等式理论（Equality Theory）：

$$ 𝐴[𝑓(𝑐−2)]=𝐴[𝑓(𝑐−2)] $$

然后应用未解释函数理论（Uninterpreted Function Theory）：

$$ 𝐴[𝑣]=𝐴[𝑣] $$

最后应用数组理论（Array Theory）可以知道该问题是可以满足的。

EUF（Equality with Uninterpreted Functions）

上面简单介绍了EUF的例子，这里给出系统的定义。

文法

$$ \begin{align*} E &::= x \ | \ c \ | \ f(E,\ \dots,\ E) \\\\
R &::= E = E \ | \ E \neq E \\\\
P &::= R \ | \ P \wedge P \end{align*} $$

示例1

$$ a = b \wedge b = c \wedge d = e \wedge b = s \wedge d = t \wedge a \neq e \wedge a \neq s $$

为了判断上面的命题是否可以满足，这里使用如下步骤：

  1. 由于 $a = b$，构建一个集合 $M = \lbrace a,\ b \rbrace$
  2. 由于 $b = c$，将 $c$ 加入 $b$ 的集合，此时 $M = \lbrace a,\ b,\ c \rbrace$
  3. 重复如上步骤，得到集合 $M = \lbrace a,\ b,\ c,\ s \rbrace, \quad N = \lbrace d,\ e,\ t \rbrace$
  4. 对于 $a \neq e$ 由于 $a,\ e$ 不在同一个集合中，显然成立。
  5. 对于 $a \neq s$ 由于 $a,\ s$ 在同一个集合中，这表明出现了矛盾。因此该命题无法满足。

可以总结出如下算法 (Naïve algorithm)：

decideEquality() {
    for (each "x = y") {
        setx = find(x);
        sety = find(y);
        union(setx, sety);
    }
    for (each "x != y") {
        setx = find(x);
        sety = find(y);
        if(setx == sety)
            return UNSAT;
    }
    return SAT;
}

示例2

$$ a = b \wedge b = c \wedge d = e \wedge b = s \wedge d = t \wedge f(a,\ g(d)) \neq f(b,\ g(e)) $$

同样使用上面的并查集算法，将函数 $f,\ g$ 和其他元素作同样的考虑。

  1. 按照上例中同样的方法，构建集合 $ M = \lbrace a,\ b,\ c,\ s \rbrace, \quad N = \lbrace d,\ e,\ t \rbrace $
  2. 由于 $d,\ e$ 在同一集合，所以 $g(d) = g(e)$，现在有集合 $F = \lbrace g(d),\ g(e) \rbrace$
  3. 由于 $a = b,\ g(d) = g(e)$，可以推知 $ f(a,\ g(d)) = f(b,\ g(e)) $，即有集合 $ K = \lbrace f(a,\ g(d)),\ f(b,\ g(e)) \rbrace $
  4. 出现矛盾，因此，该命题无法满足。

decideEquality() {
    for (each "x = y") {
        setx = find(x);
        sety = find(y);
        union(setx, sety);
    }
    while (stillChanging) {  // closure
        if ("s1 = t1", ..., "sn = tn") {
            union(f(s1, ..., sn), f(t1, ..., tn));
        }
    }
    for (each "x != y") {
        setx = find(x);
        sety = find(y);
        if(setx == sety)
            return UNSAT;
    }
    return SAT;
}

int power3(int in) {
    int i, out_a;
    out_a = in;
    for(i = 0; i < 2; i++)
        out_a = out_a * in;
    return out_a;
}

int power3_new(int in) {
    int out_b;
    out_b = (in*in)*in;
    return out_b;
}

这两个函数是等价的，但是如何证明呢？

如果要证明这两个函数等价，我们只需要证明对于任何的相同的输入，他们的返回值相等即可。由于输入的可能性太多，证明所有的输入根本不可能。所以，我们反过来，只需要证明否命题是可满足的，那么原命题就是不成立的。

现在定义一些变量：

$$ in,\ outa,\ outa1,\ outa2,\ outb $$

这些变量和上面的函数变量一一对应。

定义函数：

$$f(x,\ y) = x * y$$

现在可以得出命题：

$$ \begin{align*} P ::=\ \ \ &in = outa \\\
\wedge \ &outa1 = f(outa,\ in) \\\
\wedge \ &outa2 = f(outa1,\ in) \\\
\wedge \ &outb = f(f(in,\ in),\ in) \\\
\\
Q ::=\ \ \ &P \to (outa2 = outb) \end{align*} $$

现在只需要证明： $ \neg Q $ 是不可满足的即可。

from z3 import *

S = DeclareSort('S')
f = Function('f', S, S, S)
e, out_a, out_a_1, out_a_2, out_b = Consts('e out_a out_a_1 out_a_2 out_b', S)
P = And(out_a == e, out_a_1 == f(out_a, e), out_a_2 == f(out_a_1, e), out_b == f(f(e, e), e))
R = Implies(P, out_a_2 == out_b)
solve(Not(R))
